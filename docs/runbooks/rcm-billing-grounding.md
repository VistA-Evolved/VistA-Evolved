# RCM Billing Grounding Runbook -- Phase 39

## Overview

Phase 39 grounds the RCM module (Phase 38) in real VistA billing data. Instead
of building a separate billing engine, this phase probes the VistA sandbox,
maps available data, and exposes read-only surfaces that show live VistA data
where available and clearly-labeled "integration-pending" surfaces where the
sandbox lacks data.

## Architecture

```
Browser (RCM > VistA Billing tab)
  |
  v
GET /vista/rcm/encounters?dfn=3  <-- live via ORWPCE VISIT

  |
  v
VistA RPC Broker (port 9430)
  |
  v
WorldVistA Docker (M globals: ^AUPNVSIT, ^DIC(36), etc.)
```

## Endpoints

### Live Endpoints (real VistA data)

#### GET /vista/rcm/encounters?dfn=N
- RPC: `ORWPCE VISIT`
- Returns: Visit IEN, date/time, location, type for each PCE visit
- VistA files: 9000010 (Visit), 9000010.18 (V CPT), 9000010.07 (V POV)

#### GET /vista/rcm/insurance?dfn=N
- RPC: `IBCN INSURANCE QUERY`
- Returns: Policy ID, insurance name, group number, status
- VistA files: 36 (Insurance Company), 2.312 (Patient Insurance)

#### GET /vista/rcm/icd-search?text=X
- RPC: `ORWPCE4 LEX`
- Returns: IEN, display text, ICD code, code system
- Min 2 chars required

### Integration-Pending Endpoints

These return structured metadata about which VistA files, routines, and RPCs
to target when the data becomes available:

#### GET /vista/rcm/charges?dfn=N
- Target: ^IB(350) IB Action
- Routines: IBCF, IBCE, IBJP
- Migration: Read IB Action entries created by CPRS encounter checkout

#### GET /vista/rcm/claims-status?dfn=N
- Target: ^DGCR(399) Bill/Claims
- Routines: IBCF, IBJP, IBCE
- Migration: Claims generated by IB GENERATE CLAIM workflow

#### GET /vista/rcm/ar-status?dfn=N
- Target: ^PRCA(430) AR, ^PRCA(433) AR Transaction
- Routines: PRCAFN, PRCASER, PRCATR
- Migration: AR transactions created when claims billed and payments posted

### Capability Map

#### GET /vista/rcm/capability-map
- Returns: Full machine-readable JSON with all probed globals, RPCs, routines,
  endpoint status, and migration paths
- Source: `data/vista/capability-map-billing.json`

## Security

- All `/vista/rcm/*` routes require session auth (AUTH_RULES catch-all)
- PHI audit events:
  - `phi.rcm-encounters-view` -- encounter reads
  - `phi.rcm-insurance-view` -- insurance reads
  - `phi.rcm-charges-view` -- charge queries (pending)
  - `phi.rcm-claims-status-view` -- claims queries (pending)
  - `phi.rcm-ar-status-view` -- AR queries (pending)
  - `data.icd-search` -- ICD code searches (no PHI)
- Error sanitization: VistA internals (routine refs, file paths) stripped
- No credentials in responses

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| 401 on all endpoints | No session cookie | Login first via POST /auth/login |
| "VistA service unavailable" | Docker not running | `docker compose up -d` in services/vista |
| Empty encounters | No visit data for DFN | Try dfn=3 (has 11+ visits in sandbox) |
| ICD search returns nothing | Text too short | Min 2 characters required |
