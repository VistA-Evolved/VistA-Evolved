# Archive (salvage / reference)

This folder holds **indexes and markers** for salvage and reference content. It does not replace or move existing repo content.

## Purpose

- **Salvage:** The repo is treated as a salvage/reference archive plus a governed build platform. Existing code, prompts, and docs are preserved; they are indexed and status-marked, not deleted.
- **Reference:** Pointers and indexes here help locate legacy or superseded work and relate it to canonical docs (see `docs/canonical/source-of-truth-index.md`).

## What goes here

- Index files that list or point to archived/superseded content (e.g. by status LEGACY_REFERENCE or NOT_YET_AUDITED).
- Short READMEs or markers that explain how to use an archive index.
- **Not:** Actual product code or prompt bodies that live elsewhere; this folder is for meta-documentation only.

## Relation to other docs

- **Source-of-truth index** — `docs/canonical/source-of-truth-index.md`
- **Repo status model** — `docs/canonical/repo-status-model.md` (labels such as LEGACY_REFERENCE)
- **Governance** — `docs/governance/PROOF-AND-GOVERNANCE.md` (preserve history; archive, index, mark status)
