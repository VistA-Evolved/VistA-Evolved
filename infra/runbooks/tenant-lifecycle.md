# Tenant Lifecycle Runbook

> Covers provisioning, configuration, upgrade, and decommission of a tenant.

## Tenant Model

Each tenant gets:
- A Kubernetes namespace (`ve-tenant-<slug>`)
- An ArgoCD Application (auto-generated by ApplicationSet)
- A dedicated VistA instance (or shared, based on SKU)
- Helm values at `infra/environments/<env>/tenants/<slug>.values.yaml`

## Provisioning a New Tenant

### Step 1: Create Values File

```powershell
# Copy from template
$env = "staging"  # or dev, prod
$slug = "clinic-abc"
$src = "infra/environments/$env/tenants/demo.values.yaml"
$dest = "infra/environments/$env/tenants/$slug.values.yaml"
Copy-Item $src $dest
```

Edit the new file:
```yaml
tenantId: "clinic-abc"
api:
  env:
    TENANT_DISPLAY_NAME: "ABC Medical Clinic"
    TENANT_ID: "clinic-abc"
  image:
    tag: "latest"   # or specific version
  replicas: 2
  resources:
    requests:
      cpu: 200m
      memory: 512Mi
```

### Step 2: Commit and Push

```powershell
git add infra/environments/$env/tenants/$slug.values.yaml
git commit -m "feat(tenant): provision $slug in $env"
git push
```

ArgoCD ApplicationSet auto-discovers the new values file and creates:
- Namespace `ve-tenant-clinic-abc`
- API deployment + service
- VistA StatefulSet
- NetworkPolicies (if enabled)
- PDB (if configured)

### Step 3: Verify

```powershell
# Check ArgoCD
kubectl get application -n argocd ve-tenant-clinic-abc-$env

# Check pods
kubectl get pods -n ve-tenant-clinic-abc

# Run provisioning script (optional, for VistA routine install)
.\infra\scripts\provision-tenant.ps1 -Tenant clinic-abc
```

## Upgrading a Tenant

### Image Update
```powershell
# Single tenant
.\infra\scripts\bump-images.ps1 -Tag v1.3.0 -Environment staging -Commit
```

### Configuration Change
Edit the tenant values file, commit, and push. ArgoCD auto-syncs.

### Release Channel

Tenants can be on `stable` or `canary` channel:
```yaml
releaseChannel: canary   # Gets new versions first
```

## Decommissioning a Tenant

### Step 1: Backup
```powershell
.\infra\scripts\backup-pg.ps1 -Tenant clinic-abc
.\infra\scripts\backup-yottadb.ps1 -Tenant clinic-abc
```

### Step 2: Remove Values File
```powershell
git rm infra/environments/staging/tenants/clinic-abc.values.yaml
git commit -m "chore(tenant): decommission clinic-abc"
git push
```

ArgoCD auto-prunes the Application and its resources.

### Step 3: Verify Cleanup
```powershell
kubectl get ns ve-tenant-clinic-abc
# Should return: NotFound

.\infra\scripts\deprovision-tenant.ps1 -Tenant clinic-abc
```

## Multi-Tenant Isolation

| Layer | Mechanism |
|-------|-----------|
| Kubernetes | Namespace per tenant, NetworkPolicies |
| Database | Row-Level Security (RLS) on all tables |
| API | Tenant context from session, `SET LOCAL app.current_tenant_id` |
| ArgoCD | `ve-tenants` AppProject (no cluster resources) |
| Policy | Kyverno (resource limits, no privilege escalation) |
