# Phase 424 -- NOTES

- **Wave**: 26, Position 2 (W26 P2)
- **Type**: Infrastructure / VistA strategy
- **Risk**: Low -- new files only, single safe modification to rpcCapabilities.ts
- **Dependencies**: Phase 423 (W26 P1)

## Key Decisions
1. Runtime matrix is a static JSON document describing environments, not a
   live-generated artifact. Live data comes from rpcCapabilities.ts discovery.
2. Drift detection compares live discovery against saved baselines. Baselines
   are generated by inspect-container.ps1 and stored in data/vista/.
3. select-runtime.ps1 can optionally modify .env.local to switch between
   sandbox and distro -- requires explicit -Apply flag.
4. Domain requirements are the authoritative source for which RPCs are
   mandatory in production vs optional in development.
